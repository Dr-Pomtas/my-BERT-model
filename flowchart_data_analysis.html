🔍 詳細プロセス説明

1. 📤 データアップロード

  • CSVファイル形式での口コミデータ受付
  • ファイルサイズ制限: 16MB以下
  • セキュアファイル名処理
  • メモリ効率的な読み込み処理

2. 📋 データ形式チェック

  • 必須カラムの存在確認
  • データ型の妥当性検証
  • 欠損値の検出と処理
  • 星評価範囲チェック（1-5）

3. 📈 基本統計情報

  • 総口コミ数の集計
  • 動物病院数のカウント
  • 平均星評価の算出
  • 星評価分布の可視化

4. ⚙️ データ前処理

  • テキストクリーニング（URL除去等）
  • 特殊文字・絵文字の正規化
  • 空白文字の統一処理
  • 欠損値の補完処理

5. 🤖 感情分析実行

  • 3つのBERTモデル並列処理
  • ポジティブ/ネガティブ確率算出
  • キーワードベース分析補強
  • バッチ処理による高速化

6. 🔢 スコア計算

  • 感情スコア正規化（-2.0〜+2.0）
  • 星評価スコア変換（星評価-3）
  • 統計的な外れ値検出
  • モデル間スコア調整

7. 📋 性能評価

  • ピアソン相関係数算出
  • 平均絶対誤差（MAE）計算
  • 決定係数（R²）算出
  • 統計的有意性検定

8. 📊 可視化・出力

  • インタラクティブ散布図生成
  • モデル比較チャート作成
  • 統計サマリーテーブル
  • 結果CSVファイル出力

📋 必要なデータ形式

CSVファイルに含む必須カラム:

  • hospital_id - 動物病院の一意識別子（例：H001, H002）
  • review_text - 口コミテキスト（日本語）
  • star_rating - 星評価（1〜5の整数値）

🤖 使用する3つのBERTモデル

  • Model A: koheiduck/bert-japanese-finetuned-sentiment
  • Model B: llm-book/bert-base-japanese-v2-finetuned-sentiment
  • Model C: Mizuiro-inc/bert-base-japanese-finetuned-sentiment-analysis

各モデルの感情分析精度を比較し、最適なモデルを特定します。

⚙️ 技術スタック

  • バックエンド: Flask, PyTorch, Transformers
  • データ処理: Pandas, NumPy, Scikit-learn
  • 可視化: Plotly.js, Chart.js
  • フロントエンド: Bootstrap 5, HTML5

📈 計算式とスコアリング

感情スコア計算式

口コミスコア = (P(pos) × 2) - (P(neg) × 2)

P(pos): ポジティブ確率, P(neg): ネガティブ確率

星評価スコア変換

星評価スコア = 星評価 - 3

範囲: -2（1星）〜 +2（5星）
